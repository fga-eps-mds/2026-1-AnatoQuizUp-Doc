Metodologia — Q-Rapids (modelo de qualidade do processo)

O eixo Processo é estruturado pelo indicador estratégico Process Performance do Q-Rapids (Martínez-Fernández et al., 2019) — o mesmo modelo do eixo Produto. As métricas do board são mapeadas nas assessed metrics dos fatores Issues' Velocity (Team Throughput, Resolved Issues' Throughput, Old Issues, Bugs Ratio) e Development Speed (Commit review duration), cada uma expressa como densidade [0–1]. O indicador é a média das densidades.

Referência: MARTÍNEZ-FERNÁNDEZ, S. et al. Continuously Assessing and Improving Software Quality With Software Analytics Tools: A Case Study. IEEE Access 7, 2019 (modelo Q-Rapids, abril/2019).
dados: ZenHub (issues, sprints, eventos)GitHub (issues, PRs, labels)

Process Performance — indicador estratégico do processo

Verde ≥ 0,75 (aprovado) · amarelo ≥ 0,50 (atenção) · vermelho < 0,50 (crítico). Limiares parametrizáveis ("user defined" no Q-Rapids): lead ≤ 14 dias · idade de issue aberta ≤ 14 dias · revisão de PR ≤ 2 dias.

Dados brutos — Indicador → Fator → Assessed Metric → contagem → Fonte
